# Guion docente — Clase 2

## Inicio
“Hoy vamos a construir un agente real, útil y más cercano a un proyecto profesional.”

## Problema
“El ejemplo anterior servía para entender tools, pero todavía no se sentía real porque el clima estaba fijo.”

## Arquitectura
“El agente interpreta, usa una herramienta, consulta datos reales y luego el modelo local redacta la respuesta.”

## Memoria
“Si detecta Medellín, no responde de una vez. Primero pide el barrio o la zona.”

## Demo
“¿Va a llover hoy en Medellín?”
Luego:
“Laureles”

## Cierre
“Lo importante no es memorizar el código, sino entender la lógica: intención, herramienta, memoria, modelo y respuesta.”
